# 通用个人资料整理模板

先按用户目标与目录内实际内容选择结构。以下模板都是候选方案，不得直接套用；先展示，再由用户确认。

## 选择主维度

优先询问一个问题：这批资料日后最常通过什么方式寻找？

- **按项目/经历找**：适合当前与前期工作、学校资料、长期项目、证书、搬家资料。
- **按事件/时间找**：适合旅行照片、人生记录、活动影像、阶段性资料。
- **按主题找**：适合学习、兴趣、收藏、阅读、创作灵感。
- **按状态找**：适合桌面和下载目录，先区分活跃、归档、待确认。
- **混合结构**：顶层按状态，二层按项目/事件/主题；这是默认推荐，但不是强制。

## 模板 A：状态优先（混合目录默认候选）

```text
<archive_root>/
├─ 00_收件箱_待整理/
├─ 01_活跃与近期常用/
├─ 02_项目与经历沉淀/
├─ 03_生活记录与记忆/
├─ 04_学习_兴趣_创作资源/
├─ 05_软件_工具与数字资源/
├─ 06_私人敏感_仅索引不移动/
├─ 07_待确认与版本候选/
└─ _桌面救星/
```

适用：工作、生活、创作与下载资料混杂，且用户暂未确定更细规则。

## 模板 B：项目/经历优先

```text
<archive_root>/
├─ 01_当前项目与事务/
├─ 02_过往工作与职业经历/
├─ 03_学习与学校阶段/
├─ 04_个人项目与作品/
├─ 05_生活事务与家庭记录/
├─ 06_共享资源与工具/
├─ 07_待确认/
└─ _桌面救星/
```

适用：用户通常知道文件属于哪个机构、项目、阶段或人物关系。

## 模板 C：记忆与媒体优先

```text
<archive_root>/
├─ 01_照片与视频/
│  ├─ 按年份_事件/
│  └─ 待确认/
├─ 02_音频_录音与收藏/
├─ 03_文字记录与日记/
├─ 04_旅行_活动与人生阶段/
├─ 05_共享与导出素材/
├─ 06_待确认/
└─ _桌面救星/
```

适用：照片、视频、录音等媒体为主。禁止通过人脸、相似度或拍摄日期自动删除、合并、重命名；默认保持原文件名和原时间信息。

## 模板 D：创作与兴趣优先

```text
<archive_root>/
├─ 01_进行中作品与项目/
├─ 02_已完成作品与发布物/
├─ 03_设定_素材_灵感/
├─ 04_学习笔记与课程/
├─ 05_兴趣收藏与参考/
├─ 06_工具_模板与资源/
├─ 07_待确认与版本候选/
└─ _桌面救星/
```

适用：写作、绘画、摄影、音乐、代码、手作等个人创作或兴趣资料为主。

### 创作资产规则

- 对进行中作品，默认“仅索引、不移动”；只有用户点名后才移动。
- 对文件夹，优先整体移动，不拆分正文、设定、素材、版本。
- 对 `v1`、`final`、日期、草稿、备份等版本组，默认全部保留并放在同一语境下。
- 不根据文件名猜测哪个版本“更正确”。

## 模板 E：轻量下载目录

```text
<archive_root>/
├─ 01_需要处理/
├─ 02_已归档资料/
├─ 03_安装包与工具/
├─ 04_临时导出与待确认/
└─ _桌面救星/
```

适用：下载目录。仅整理来源明确、用途明确且用户确认的文件；压缩包、安装包、浏览器导出、临时文件默认进入待确认或保留原位。

## 语境命名规则

- 使用稳定、可搜索、对用户有意义的名称，例如：`2025_某地旅行`、`大学_毕业设计`、`前公司_项目A`、`小说_作品名`。
- 名称中只在用户已有清晰标签时加入年份；不要根据文件日期擅自命名事件。
- 不改原文件名；目录名是分类标签，不是对文件内容的替代说明。
- 避免超过两层的机械嵌套；遇到不确定项使用 `待确认`，不要创建猜测性的细目录。

## 需要单独确认的类别

在以下情况中，不得使用模板直接移动：

- 当前正在使用或过去 30 天频繁修改的资料。
- 个人照片原片、视频、音频、聊天记录、日记。
- 小说、文章、设计、代码、研究等未发布或有版本链的创作资产。
- 证件、金融、医疗、法律、账号恢复或私人通信资料。
- 内容未知的压缩包、加密文件、磁盘映像、应用程序或大型文件夹。
